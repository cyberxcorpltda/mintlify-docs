Cria um monitoramento processual para o número e tribunal informados.

O campo `court_code` é obrigatório porque um mesmo número deve ser validado contra o tribunal correto.

## Corpo da requisição

<ParamField body="process_number" type="string" required>
  Número do processo no padrão CNJ.
</ParamField>

<ParamField body="court_code" type="string" required placeholder="tjsp">
  Sigla/código público do tribunal DataJud, como `tjsp`, `tjba`, `trf1`, `tst` ou `stj`.
</ParamField>

<ParamField body="name" type="string">
  Nome interno do monitoramento.
</ParamField>

<ParamField body="frequency_id" type="string" required placeholder="daily">
  ID da frequência retornada em [Configuração](/api-reference/monitoring/process/config).
</ParamField>

<ParamField body="notification_email" type="string" required>
  E-mail que receberá os alertas do monitoramento.
</ParamField>

## Exemplo de corpo

```json
{
  "process_number": "0000001-77.2026.8.26.0000",
  "court_code": "tjsp",
  "name": "Processo cliente exemplo",
  "frequency_id": "daily",
  "notification_email": "alertas@example.com"
}
```

## Fluxo recomendado

1. Use [Detectar tribunal](/api-reference/monitoring/process/detect-court) para obter `court_code`.
2. Use [Validar processo](/api-reference/monitoring/process/validate) para confirmar que o processo foi encontrado no tribunal.
3. Crie o monitoramento com a frequência desejada.
4. Acompanhe eventos em [Detalhar monitoramento](/api-reference/monitoring/detail).
